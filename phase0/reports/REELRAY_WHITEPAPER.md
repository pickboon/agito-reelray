## 一、执行摘要

**极光 ReelRay** 是 Agito 科技旗下的 AI 短剧出海生产平台，定位为「专注角色锁定的 AI 短剧出海生产工具」。

**Phase 0 核心结论：技术可行性确认通过。** 在 7 个维度、58 条测试用例中，HappyHorse API 综合成功率 96.6%（56/58），其中唯一失败案例为阿里云内容安全审核拦截，非生成能力问题。**关键判据——多镜头叙事连贯性——已通过 r2v 参考图锚定方案验证，确认有效。**

**建议：立即进入 Phase 1 平台脚手架搭建。**

### 2.3 1.0 vs 1.1 对比要点

7 对相同 prompt + seed 的对比测试（中性、侧脸、远景、走路、金色、月光、霓虹），全部成功。1.1 在视觉质感（去除油光感/过度锐化）和指令遵循方面明显优于 1.0，且新增 r2v 参考图锚定能力——这是角色锁定的关键技术路径。

### 2.4 多镜头叙事连贯性（核心判据）

**场景**：晨光咖啡馆，5 个连续镜头（进场→近景→窗边→喝咖啡→退场）

| 组别 | 方案 | 结果 | 角色一致性 |
|------|------|------|-----------|
| A 组 | 纯文本 t2v | 5/5 成功 | ⚠️ 不可靠，跨镜头面部漂移明显 |
| B 组 | r2v 参考图锚定 | 5/5 成功 | ✅ 明显优于 A 组，角色锁定有效 |

**结论：纯 Prompt 描述不足以保证跨镜头角色一致性，r2v 参考图锚定是可行的角色锁定方案。**

## 三、API 运行特征

### 3.1 性能画像

| 指标 | 数值 |
|------|------|
| 平均生成耗时 | 5 秒视频约 140-170 秒（1080p） |
| 最快生成 | 8 秒（简单场景） |
| 最慢生成 | 210 秒（复杂光照） |
| 输出分辨率 | 1080p / 16:9 |
| 并发能力 | 2 路并行稳定，无明显的队列拥堵 |
| 网络异常 | 偶发 ConnectionResetError（~3%），重试可恢复 |

### 3.2 已确认的 API 细节

- **模型名**: `happyhorse-1.0-t2v` / `happyhorse-1.1-t2v` / `happyhorse-1.1-r2v`
- **端点**: `POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis`
- **size 参数格式**: `1280*720`（星号，不是 x）
- **duration 参数**: 整数（秒），非浮点
- **异步模式**: 提交→轮询，无 callback/webhook
- **r2v 参数**: `media` 为对象数组，每项需 `url` 字段（图片 URL 或 base64 data URL），最多 9 张参考图
- **水印**: 所有输出文件名含 `_refiner_watermark.mp4`，API 无去水印参数（商用版可协商无水印）

### 3.3 成本模型（基于公开信息）

|------|--------|--------|
| 720p | ¥0.9/秒 | ¥0.44/秒 |

- 720p: 120 秒 × ¥0.44 × 1.3 ≈ **¥69**
- 1080p: 待确认，预计 ¥100-150

## 六、技术架构建议

### 6.1 技术栈（复用极智平台）

- Next.js 15.5 + React 19 + TypeScript + TailwindCSS 4
- Shadcn/ui 组件库
- Supabase（认证 + 数据库）
- Cloudflare R2（视频存储）
- Stripe（支付）
- CF Pages（部署）

### 6.2 核心数据模型（6 张表）

| 表 | 说明 |
|----|------|
| `projects` | 短剧项目 |
| `characters` | 角色定义 + 参考图 |
| `episodes` | 集 |
| `shots` | 镜头（含 t2v/r2v 参数 + 生成结果） |
| `generation_logs` | 生成日志（可审计） |
| `subscriptions` | 订阅/支付记录 |

### 6.3 视频生成流水线

```
剧本 → LLM 拆解分镜 → 角色参考图生成 → r2v 锚定生成 → 多语种版本 → 质量评分 → 分发格式导出
```
