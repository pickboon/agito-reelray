# Phase 0 — 角色锁定 API 成功率验证

## 目标
在 2 周内，用 <¥500 现金成本，验证 HappyHorse API 的角色锁定成功率是否 ≥ 40%。

## 验证维度

| 编号 | 测试场景 | 说明 | 关键度 |
|------|---------|------|--------|
| T1 | 面部一致性 | 同一角色在多段生成中面部特征保持一致 | ⭐⭐⭐ |
| T2 | 多镜头连贯 | 不同景别/角度切换时角色不漂移 | ⭐⭐⭐ |
| T3 | 光影统一 | 同一场景下光照方向、色调一致 | ⭐⭐ |
| T4 | 衣着头饰 | 服装颜色、款式、配饰在跨场景中保持一致 | ⭐⭐⭐ |
| T5 | 背景空间 | 同一场景下背景元素、空间关系不漂移 | ⭐⭐ |

## 测试方法
1. 设计统一角色描述（character anchor）
2. 每个维度生成 10 段视频
3. 人工 + LLM 双重评分（1-5 分）
4. 成功率 = (评分 ≥ 4 的段数) / 总段数

## 决策节点
- ≥ 40% 成功率 + ≥ 2 家客户意愿 → Phase 1
- 30-39% 成功率 → 延期一周攻坚
- < 30% 成功率 → 暂停主产品线
