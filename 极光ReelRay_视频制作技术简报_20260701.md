## 第一块：API 接口与技术边界

### 基本信息

| 项 | 值 |
|----|-----|
| 模型名称 | `happyhorse-1.1-t2v`（文生视频）/ `happyhorse-1.1-r2v`（图生视频） |
| 提供方 | 阿里云 DashScope |
| 提交端点 | `POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis` |
| 轮询端点 | `GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}` |
| 工作模式 | 提交 → 拿 `task_id` → 轮询 → 拿视频 URL。**无 callback/webhook，纯轮询** |
| 认证 | Header `Authorization: Bearer {API_KEY}` + `X-DashScope-Async: enable` |
| SDK 封装 | `lib/adapters/happyhorse.ts` — HappyHorseAdapter 类，实现 IVideoModelAdapter 接口 |
| 类型定义 | `lib/adapters/types.ts` — GenerateParams / SubmitResult / PollResult / TaskStatus / CostEstimate |

### t2v 请求体（文生视频）

```json
{
  "model": "happyhorse-1.1-t2v",
  "input": {
    "prompt": "Cinematic wide shot of..."
  },
  "parameters": {
    "size": "720*1280",
    "duration": 15
  }
}
```

### r2v 请求体（图生视频 - 角色锁定核心通路）

```json
{
  "model": "happyhorse-1.1-r2v",
  "input": {
    "prompt": "Medium close-up, same character...",
    "ref_img_url": "https://r2-public-url/anchors/character_xxx.jpg"
  },
  "parameters": {
    "size": "720*1280",
    "duration": 10
  }
}
```

### 完整参数表

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | string | 是 | `happyhorse-1.1-t2v` 或 `happyhorse-1.1-r2v` |
| `input.prompt` | string | 是 | 文本提示词，中英文均可 |
| `input.ref_img_url` | string | r2v 必填 | 参考图 URL（仅 r2v 模式） |
| `parameters.size` | string | 是 | **格式 `"720*1280"`（星号，不是 `x`）** |
| `parameters.duration` | int | 是 | 整数秒，非浮点，非字符串 |
| `parameters.seed` | int | 否 | 用于复现结果 |

### 能力边界

| 维度 | 上限/范围 |
|------|----------|
| 最大单条时长 | **15 秒** |
| 支持画幅比 | `9:16`（竖屏 720×1280）/ `16:9`（横屏 1280×720）/ `1:1`（方形 720×720） |
| 最大参考图数 | **9 张**（r2v 模式 `input.media` 数组，最多 9 个 `{url}` 对象） |
| 原生音频 | ✅ 支持，7 语言唇同步：zh / en / ja / ko / es / fr / pt |
| 输出分辨率 | 720p / 1080p（H.264, ~9.5Mbps） |
| 并发 | 测试 Key 2 路并行稳定，无队列拥堵 |
| 生成速度 | 5s 视频约 **140-210s**；10s 视频约 **280-400s**；中位数 ~170s（1080p/5s） |

### ⚠️ 已知坑（全部来自 Phase 0 实测，共 7 个）

| # | 坑 | 详情 | 对策 |
|---|-----|------|------|
| 1 | **size 必须是星号** | `"720*1280"` 不能写成 `"720x1280"`，否则静默失败 | 适配层已处理（`aspectToSize` 方法） |
| 2 | **duration 必须整数** | 浮点数 `10.5` 会被截断为 `10` | 入参校验 |
| 3 | **水印** | 所有输出文件名含 `_refiner_watermark.mp4`，API 无去水印参数 | 商用版可协商无水印，现阶段样片保留水印即可 |
| 4 | **OSS URL 有时效性** | 轮询返回的视频 URL 是临时的，过期无法下载 | 拿到后必须 **立即下载** 到 R2/本地 |
| 5 | **内容安全拦截** | 面部近景特写有 ~3% 概率触发阿里云 Green Net 审核，直接返回失败 | Prompt 加 `portrait photography style, editorial fashion` 安全锚点词 |
| 6 | **网络闪断** | 偶发 `ConnectionResetError`（~3%） | 代码已内置 3 次自动重试 + 指数退避（`lib/engine/anchor.ts`） |
| 7 | **无 callback** | 不能等服务端推送完成通知 | 前端/后端驱动轮询，适配层 `poll()` 每 5s 查一次，最多 40 次 |

### ✅ 模型擅长项

- **光影渲染** — 日落、阴天、月光、正午、霓虹全部正确，这是短剧质感的基石
- **衣着头饰保持** — 同一套衣服 + 发型在不同镜头里保持高度一致
- **背景空间复现** — 提示词描述的室内外场景元素可以跨镜头重现
- **欧美面孔** — 生成质量好于亚洲面孔
- **多角度镜头** — 远景/中景/近景/特写/过肩均可用

### ❌ 模型翻车点

| 翻车场景 | 严重程度 | 详情 | 应对 |
|---------|---------|------|------|
| 纯 Prompt 跨镜头角色不一致 | 🔴 致死 | A 组 5/5 生成成功，但面部漂移明显，观众能认出来不是同一个人 | **必须走 r2v 参考图锚定** |
| 近景面部特写 | 🟡 偶发 | 触发了阿里云内容安全审核（概率 ~3%） | Prompt 加安全锚点词 |
| 亚洲面孔偏欧美化 | 🟡 常见 | Prompt 不强调 "East Asian" 时，默认生成偏欧美 | Prompt 中明确种族特征 |
| 多角色同框 | 🟠 未充分测试 | Phase 0 边界用例通过了但仅 3 条 | 样片暂不碰复杂多角色 |
| 微表情/嘴型精度 | 🟠 未覆盖 | API 声称支持 7 语言唇同步，但实际效果未验证 | 有条件时专门测试 |
| 大场景/特效 | 🔴 不擅长 | HappyHorse 不是特效模型 | 样片避开此方向 |

### 角色一致性上限（核心判据）

- **已验证**：同一角色 r2v 锚定可追 **5 个连续镜头**（进场→近景→窗边→喝咖啡→退场），面部一致性明显优于纯 t2v
- **方案**：第 1 镜用 t2v 生成首帧 → 提取首帧为参考图 → 第 2-N 镜全部用 r2v 锚定同一张参考图
- **30s 样片可实现**：3-4 个镜头，每个 7-10s，全程同一张脸

### 动作戏

- Phase 0 未专门覆盖动作戏测试
- 边界用例中"动态模糊"一条通过（TE_01），但远不能代表格斗/打斗的稳定性
- **建议样片中专门做一条动作戏来摸上限** — 如果动作戏能撑住，这是对竞品的巨大差异化

## 第三块：样片目标

### 核心目标

**证明「这模型做出来的短剧品质能接商业单」。**

不是跑通流程（Phase 0 已经证明 96.6% 成功率），而是让潜在用户看完样片后产生一个念头：「这画质 + 这人脸一致性，我能用来出海赚钱。」

### 目标受众

| 受众 | 优先级 | 他们要看什么 |
|------|--------|------------|
| 出海短剧创作者 | P0 | "这工具能帮我稳定输出角色不换脸的内容" |
| 平台方 / 投资人 | P1 | "技术可行 + 品质达到商业化门槛" |
| 早期种子客户 | P1 | "值得试，比我自己用通用模型做省心" |

### 样片时长

**5 秒不行。** 一个 5 秒镜头看不出连贯性，看不出角色锁定——这两项恰恰是 ReelRay 唯一核心卖点。

| 类型 | 时长 | 镜头数 | 理由 |
|------|------|--------|------|
| Banner 视频 | **15s**（单条 API 上限） | 3-4 个快切 | 同一角色进场→近景→情绪→退场，3 秒内让观众看懂"角色没变" |
| 精品样片 | **30s**（2 条拼接） | 4-5 个镜头 | 能讲一个小型叙事闭环，起承转合完整，让人产生"我能用这个"的决策 |
| 模板库 preview | **10-15s** | 2-3 个镜头 | 展示该模板典型风格，hover 播放 |

**30s 实现方式**：HappyHorse 单次调用上限 15s，30s 精品样片 = 2 条 API 调用 → API 返回视频文件 → 本地/编辑器拼接。避免用 API 合成，直接源文件剪辑更快。

### 功能特性强调（不是纯综合素质展示）

每一条 30s 样片至少证明以下 2 项：

| 特性 | 怎么展示 | 重要程度 |
|------|---------|---------|
| **角色一致性** | 同一角色在不同景别/角度/光照下的面部不漂移 | 🔴 必须（唯一核心卖点） |
| **光影统一** | 多镜头切换时光照色调不跳变 | 🟡 重要 |
| **动作稳定性** | 如果战神赛道能跑通格斗/打斗，这是巨大加分项 | 🟢 有就突出 |
| **多语种唇同步** | 同一条片子出中/英双语版本，直接证明出海能力 | 🟢 有预算就做 |

**不要碰的**：大场景特效 / 复杂多角色同框 / 纯风景空镜（跟角色锁定无关，浪费样片机会）

### 竞品参照（聚焦 AI 短剧作品本身）

| 参照作品 | 平台 | 为什么参照 | 样片目标 vs 它 |
|---------|------|-----------|--------------|
| **《山海奇镜之劈波斩浪》** | 快手可灵 | 2024 首部 AI 奇幻短剧，被公认是那时"AI 短剧可接受质量线" | **画质和角色一致性必须明显超过它** |
| **《兴安岭诡事》** | 快手可灵 | AI 悬疑短剧，暗调气氛和光影处理出圈 | **暗调质感对标它，角色一致性超过它** |
| **ReelShort 真人短剧** | ReelShort App | 出海短剧品质标杆（真人拍摄） | **不需要达到真人级，但镜头语言往这个方向靠** |

**对标一句话**：样片品质 = 角色一致性超过《山海奇镜》+ 镜头质感接近 ReelShort。

> 注：不参照 Runway / Pika / Kling 等通用工具的宣传片 — 产品逻辑不同，没有参考价值。

### Banner 视频具体要求

| 维度 | 规格 |
|------|------|
| 画幅 | **`9:16` 竖屏**（720×1280，对齐 TikTok/ReelShort 主流格式） |
| 时长 | **15 秒**（单条 API 上限） |
| 内容 | 同一个角色的 3-4 个镜头快切：进场→近景（表情）→高光动作→退场 |
| 风格 | 暗调 + 电影级打光（赛博青 `#00F0FF` + 品牌金 `#FACC15` 色调，不要太亮太日系） |
| 音频 | Banner 无声自动循环播放，不需要配乐/配音 |
| 核心目的 | 观众 3 秒内看懂一句话「这个角色，从头到尾不会变脸」 |

**为什么选这个方向**：竞品落地页 banner 基本都是宣传片式剪辑。ReelRay 差异化是 r2v 角色锁定，banner 就应该直接展示成果——同一张脸在不同镜头里始终一致。认知门槛最低，3 秒直达价值主张。
